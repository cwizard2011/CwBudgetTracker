## Project Overview

React Native mobile app (iOS & Android) for budget and loan tracking. Uses Firebase (Firestore + Storage + Auth) as the backend, with offline-first local storage via AsyncStorage.

## Commands

```bash
# Start Metro bundler
npm start

# Run on Android/iOS
npm run android
npm run ios

# Lint
npm run lint

# Tests
npm run test
npx jest --testPathPattern="<file>"   # Run a single test file

# Android release builds
npm run android:bundle:release        # AAB for Play Store
npm run android:assemble:release      # APK

# iOS release builds (see RELEASE.md for full instructions)
```

## Architecture

### State Management

All state lives in four React Contexts under [src/context/](src/context/):

- **BudgetContext** — budget CRUD, recurring budget generation, series edit logic
- **LoanContext** — loan CRUD, counterparty tracking, payment recording
- **CategoryContext** — user-defined categories (persisted to AsyncStorage)
- **SettingsContext** — theme, locale, currency (persisted to AsyncStorage)

There is no Redux or Zustand. Components import context hooks directly.

### Offline-First Sync

[src/services/LocalStorage.ts](src/services/LocalStorage.ts) is the source of truth for all app data. It wraps AsyncStorage and maintains a **pending mutations queue** for operations made while offline.

[src/services/SyncService.ts](src/services/SyncService.ts) watches network connectivity (NetInfo) and:

1. **pushPending()** — flushes queued mutations to Firestore
2. **pullRemote()** — fetches all remote records
3. **mergeRecordsById()** — picks winner by `updatedAt` timestamp; never overwrites local data with an empty remote response

### Recurring Budgets

Recurring budget logic is entirely in `BudgetContext`. Key concepts:

- A recurring **rule** budget has `recurring` set to a period (weekly/monthly/quarterly/annually) and `isAutoGenerated: false`
- **Occurrences** are auto-generated with `isAutoGenerated: true` and share a `recurringGroupId`
- Edits apply to **single**, **this and future**, or **all** in the series — this detaches modified occurrences from the series

### Navigation

Stack navigator rooted at `HomeScreen`. The old tab navigator (`Sections` screen) is still present but the app now uses a card-based home. Screens live in [src/screens/](src/screens/).

### Theming

[src/theme/colors.ts](src/theme/colors.ts) exports a mutable `Colors` object. When the user changes theme, `applyTheme()` mutates this object in place — components that reference `Colors` directly pick up changes on next render without a full re-render tree.

Four palettes: `light`, `dark`, `darkDim`, `darkGray`. System theme is also supported.

### Backup / Restore

Two backup strategies:

- **[BackupService.ts](src/services/BackupService.ts)** — exports/imports a local JSON file (Downloads or Documents folder)
- **[GoogleDriveBackupService.ts](src/services/GoogleDriveBackupService.ts)** — OAuth 2.0 upload/restore to hidden app data folder on Google Drive

OAuth client IDs (debug vs release) are configured in [src/config/googleDrive.ts](src/config/googleDrive.ts) — use the **Web** client ID, not the Android one.

### Internationalization

[src/utils/i18n.ts](src/utils/i18n.ts) with ~150 keys across EN, PT, FR, ES, DE. Use the `useI18n()` hook in components.

### Models

[src/models/](src/models/) defines TypeScript interfaces for `Budget`, `Loan`, and `Invoice`. All timestamps are epoch milliseconds; budget periods use `YYYY-MM` format; dates use `YYYY-MM-DD`.
